# 刷题答案第一卷：C++ 与算法

## C++ 语言与对象模型

### 1. 静态多态和动态多态

静态多态在编译期决定调用目标，典型形式是函数重载、运算符重载和模板；动态多态通过虚函数在运行期按对象真实类型分派。前者没有虚调用开销但需要编译期知道类型，后者允许通过基类接口操作不同派生类型。

### 2. 虚函数调用过程

典型 ABI 会在多态对象中保存 vptr，vptr 指向该动态类型的虚函数表。通过指针或引用调用虚函数时，程序读取 vptr、按固定槽位取出函数地址，再进行间接调用；具体布局属于 ABI 实现细节，不是 C++ 标准强制规定。

### 3. 虚析构函数

通过基类指针 `delete` 派生对象时，虚析构保证先调用派生析构再调用基类析构。若基类析构非虚，这种删除属于未定义行为，派生资源可能不释放。

### 4. 构造函数与虚函数

构造开始前还没有一个完整的动态对象，调用哪个构造函数由创建表达式直接确定，所以构造函数不能虚。析构需要从基类接口进入并按动态类型清理完整对象，因此可以且常常应该虚。

### 5. 构造析构阶段的虚调用

构造基类部分时，虚调用只分派到当前正在构造的类，不会进入尚未构造的派生类；析构时同理，不会进入已经销毁的派生部分。依赖派生状态的虚调用不应放在构造或析构函数中。

### 6. overload、override 与 hiding

Overload 是同一作用域中同名函数具有不同参数列表；override 是派生类用相同签名覆盖基类虚函数；hiding 是派生类声明同名成员后隐藏基类的整组同名函数。`using Base::Func` 可以重新引入被隐藏的重载。

### 7. 纯虚函数、抽象类与接口

含至少一个未实现纯虚函数的类是抽象类，不能直接实例化。C++ 没有独立的 interface 关键字，通常用只有纯虚函数和虚析构的抽象类表达接口；纯虚函数仍可在类外提供定义，但类依然抽象。

### 8. 多继承中的 vptr

数量由 ABI 和基类布局决定，常见实现中每个需要独立多态子对象的基类部分都有自己的 vptr。不能简单回答“多继承一定两个”，应根据哪些基类具有虚函数以及编译器 ABI 分析。

### 9. 菱形继承与虚继承

普通菱形会包含两份共同基类子对象，导致成员二义性和重复状态。虚继承让最派生对象只保存一份虚基类子对象，并通过额外布局信息定位它；代价是对象布局和访问更复杂，虚基类由最派生类负责构造。

### 10. 空类大小

不同完整对象必须能拥有不同地址，所以空类通常占 1 字节。空基类优化和 `[[no_unique_address]]` 允许某些子对象不额外占空间，但不能把“空类永远一字节”当作所有布局的绝对规则。

### 11. 类大小计算

先按声明顺序放置非静态成员并满足各自对齐，再考虑基类子对象、常见 ABI 下的 vptr、成员间填充和末尾填充；最终大小是最大对齐的整数倍。静态成员不计入实例，虚函数代码和 vtable 本体也不在每个实例内。

### 12. 内存对齐

对齐让 CPU 以适合的边界访问数据，并使数组中每个元素保持相同对齐。`alignof(T)` 给出类型对齐要求，`sizeof(T)` 包含填充且必须是该对齐的整数倍，因此 `T[]` 中相邻元素仍正确对齐。

### 13. static 成员与对象大小

静态数据成员属于类而不属于某个对象，程序中通常只有一个定义，所以实例只保存非静态成员和必要的布局信息。静态成员函数也没有隐式 `this`。

### 14. override 与 final

`override` 让编译器验证函数确实覆盖了基类虚函数，可发现签名和 const 错误。`final` 可禁止类继续派生，或禁止某个虚函数继续覆盖。

### 15. RTTI、typeid 与 dynamic_cast

RTTI 为多态类型提供运行时类型信息。`typeid` 查询表达式类型信息；`dynamic_cast` 沿多态继承关系做受检转换，指针失败返回 null，引用失败抛 `std::bad_cast`。它通常依赖编译器生成的类型描述和继承关系数据。

## 内存与生命周期

### 16. 进程地址空间

典型布局包括代码段、只读数据、已初始化数据、BSS、堆、内存映射区和线程栈。实际地址、增长方向和分段方式由操作系统、装载器与 ABI 决定，ASLR 会进一步随机化。

### 17. 栈与堆

栈按调用帧自动管理，分配快、局部性好但容量有限；堆支持动态大小和跨作用域生命周期，分配器需要维护元数据并可能产生碎片。现代程序应通过 RAII 管理堆资源，而不是裸 new/delete 配对。

### 18. `new int` 与 `new int()`

`new int` 默认初始化标量，所得值不确定；`new int()` 值初始化，得到 0。类类型是否调用哪个构造函数则由其构造函数定义决定。

### 19. new/delete 与 malloc/free

new 先申请存储再构造对象，失败默认抛异常；delete 先析构再释放。malloc/free 只管理原始字节并返回 `void*`，不调用构造析构。二者必须各自配对，不能交叉释放。

### 20. placement new

placement new 在调用者提供的地址上构造对象，不负责申请内存。对象结束时需显式调用析构，再由原存储的所有者释放内存；常用于容器、内存池和 variant 类实现。

### 21. delete 如何知道大小

C++ 标准规定语义而不规定元数据布局。实现可能从分配器块头、编译器传入的 sized delete 参数或数组 cookie 获取信息；`delete[]` 还需知道元素数量以逐个析构，因此绝不能用标量 delete 释放 new[]。

### 22. 浅拷贝与 double free

浅拷贝只复制资源地址，两个对象随后认为自己都拥有同一资源，析构时可能重复释放。解决方法是定义深拷贝、转移所有权、共享所有权，或直接使用标准 RAII 类型。

### 23. Rule of Three/Five/Zero

管理资源的类若需要自定义析构、拷贝构造或拷贝赋值，通常三者都要审视；C++11 后还要审视移动构造和移动赋值，形成 Rule of Five。最好把资源交给标准类型，让编译器生成全部特殊成员，即 Rule of Zero。

### 24. RAII

RAII 把资源获取绑定到对象构造，把释放绑定到析构。局部对象即使因异常或提前 return 离开作用域也会清理，因此文件句柄、互斥锁和动态内存都可获得确定性释放。

### 25. unique_ptr 所有权

unique_ptr 表达唯一所有权，复制会产生两个所有者，所以被禁用；使用 `std::move` 转移后，源指针变空。自定义删除器可管理文件、句柄或特殊分配器资源。

### 26. shared_ptr 控制块

控制块通常保存强引用计数、弱引用计数、删除器、分配器以及对象定位信息。shared_ptr 对象保存被指向地址和控制块引用，因此别名构造时这两个地址可以不同。

### 27. make_shared

make_shared 通常把对象和控制块放在一次分配中，减少分配次数并改善局部性。代价是只要 weak_ptr 仍保留控制块，合并分配的整块存储可能无法归还；它也不适合需要特殊 delete 表达式的场景。

### 28. weak_ptr 与循环引用

weak_ptr 观察控制块但不增加强计数。循环关系中的一侧改为 weak 后不会阻止强计数归零；使用前调用 `lock()` 原子地取得临时 shared_ptr，失败则对象已销毁。

### 29. shared_ptr 的线程安全

不同 shared_ptr 实例共享的控制块引用计数操作是线程安全的，但它们指向的对象并不会因此线程安全。同一个 shared_ptr 变量被多个线程无同步地读写也有数据竞争，需使用锁或原子 shared_ptr 接口。

### 30. 值类别

lvalue 有稳定身份；prvalue 主要表示用于初始化或计算的纯值；xvalue 有身份但资源可被复用。rvalue 是 prvalue 与 xvalue 的集合。实际判断应看表达式类别，而不是简单看“有没有名字”。

### 31. `std::move`

`std::move` 本质是转换为右值引用的 cast，不执行数据搬运。之后是否移动取决于重载解析是否选择移动构造/赋值，以及目标类型的实现。

### 32. noexcept 移动构造

vector 扩容需要在异常时保持原容器有效。如果移动可能抛异常而拷贝可用，容器常选择拷贝；声明正确的 noexcept 能让容器安全地使用移动。不能为了性能虚假标记，否则抛出时会 terminate。

### 33. RVO、NRVO 与 move

RVO/NRVO 允许或要求直接在调用方存储位置构造返回对象，省掉移动。返回具名局部变量时通常直接 `return obj;`，显式 `std::move(obj)` 反而可能阻止 NRVO；无法消除时编译器再尝试移动。

### 34. 四种 cast

static_cast 用于编译期可验证的常规转换；dynamic_cast 用于多态继承的运行时受检转换；const_cast 只改变 cv 限定，修改原本 const 对象仍是未定义行为；reinterpret_cast 做低层位模式或地址解释，最危险且不可移植。

### 35. Lambda 异步引用风险

引用捕获或捕获裸 this 不延长对象生命周期，异步回调执行时原对象可能已经销毁。应按所有权选择值捕获、shared/weak 所有权、取消任务，或确保回调生命周期严格短于被引用对象。

## const、模板与编译

### 36. 三种 const 指针

`const int* p` 是指向常量的指针，能换目标不能改值；`int* const p` 是常量指针，不能换目标但能改值；`const int* const p` 两者都不能改。

### 37. 顶层与底层 const

顶层 const 修饰对象本身，按值复制时通常被丢弃；底层 const 修饰所指向或引用的对象，转换时必须保留 const 安全。按值参数的顶层 const 不构成新重载，指针指向类型的底层 const 可以。

### 38. const 成员函数的 this

在 `T::f() const` 中，this 的语义近似 `T const* const`：指针本身不可重新赋值，且默认不能通过它修改非 mutable 成员。const 也是成员函数签名的一部分，可与非 const 版本重载。

### 39. 只按返回类型重载

不能，因为调用点在选择函数时不一定使用返回值，编译器无法仅凭返回类型完成重载解析。

### 40. `int` 与 `const int` 参数

按值参数的顶层 const 不属于函数类型，两者是同一签名，不能重载。`int*` 与 `const int*` 的底层 const 不同，可以重载。

### 41. 模板定义为何放头文件

编译器在实例化点需要看到完整定义，才能为具体模板参数生成代码。也可以把定义放 cpp，但必须显式实例化所需类型，否则其他翻译单元会缺少定义。

### 42. 特化与偏特化

全特化为一组完整模板参数提供独立实现；偏特化只固定一部分模式，并在匹配类型时选用。函数模板不支持偏特化，通常用重载替代。

### 43. SFINAE

模板候选在替换参数时若形成无效类型或表达式，该候选被移出重载集合，而不是让整个编译失败。只有发生在即时上下文中的替换失败才适用这一规则。

### 44. Concept

Concept 用命名约束直接描述模板参数需求，参与重载排序，并给出更清晰的诊断。它比层层 enable_if 更易读，但底层仍是编译期约束与候选筛选。

### 45. 构建流程

预处理展开 include、宏和条件编译；编译把每个翻译单元变成汇编或中间表示；汇编生成目标文件；链接解析跨文件符号、重定位地址并组合库生成可执行文件。

### 46. 编译错误与链接错误

语法、类型和模板实例化错误通常发生在编译阶段。声明存在但定义缺失、重复定义或库未链接等问题通常在链接阶段暴露。

### 47. ODR

One Definition Rule 约束实体在程序中的定义数量。头文件中的非 inline 普通函数被多个翻译单元包含后产生多个外部定义；模板、类内定义和 inline 函数允许多处相同定义，由链接器合并。

### 48. inline、static 与匿名命名空间

inline 的核心是允许跨翻译单元出现相同定义，不保证实际内联。命名空间作用域 static 和匿名命名空间都提供内部链接，使实体仅在当前翻译单元可见；现代 C++ 更推荐匿名命名空间组织内部实体。

### 49. 宏、constexpr 与 inline function

宏只是文本替换，没有类型和作用域，应主要用于条件编译。constexpr 用于可在编译期求值且类型安全的值或函数；inline 函数适合头文件中的短定义，并遵循语言的类型检查。

### 50. 大端与小端

大端把最高有效字节放在低地址，小端相反。可将多字节整数的地址安全地观察为 `unsigned char*` 检查第一个字节；网络协议通常明确使用网络字节序，不能依赖主机端序。

## STL 与数据结构

### 51. vector、list、deque

vector 连续存储，随机访问和遍历优秀，但中间插删移动元素；list 节点分散，已知位置插删稳定，但随机访问慢且缓存差；deque 分块连续，支持两端常数时间扩展，随机访问可用但局部性弱于 vector。

### 52. vector 遍历速度

连续元素使硬件预取和 cache line 利用率高，迭代只做简单地址递增。链表即使同为 O(n)，每个节点仍可能触发指针追踪和缓存未命中。

### 53. vector 扩容

容量不足时申请更大连续区，移动或拷贝元素，销毁旧元素并释放旧存储，因此引用、指针和迭代器失效。标准不规定固定增长倍数，只要求复杂度性质；倍数由实现决定。

### 54. reserve 与 resize

reserve 只保证容量至少达到指定值，不改变 size 或构造新元素；resize 改变元素数量，扩大时会构造元素，缩小时会析构元素。

### 55. vector 迭代器失效

发生重分配时全部引用、指针和迭代器失效。没有重分配的尾部 push 通常保留旧元素引用；insert/erase 会使操作位置及其后的迭代器失效。应查对应操作的标准保证而不是靠经验猜。

### 56. emplace_back 与 push_back

emplace_back 用参数在容器尾部直接构造元素，可能避免一个临时对象；如果已经有 T 对象，push_back 的移动同样高效。emplace 还可能选择意外构造函数，不能认为永远更快或更安全。

### 57. map 与红黑树

红黑树用较弱的平衡约束换取搜索、插入和删除都为 O(log n)，且旋转次数相对可控，适合关联容器。标准只规定复杂度和行为，不强制实现必须是红黑树。

### 58. 红黑树与 AVL

红黑树保证从根到叶的最长路径不超过最短路径两倍，因此高度 O(log n)。AVL 平衡更严格，查询常更快但更新可能需要更多调整；红黑树更适合更新和查询混合的通用容器。

### 59. unordered_map

哈希函数把键映射到桶，桶内再通过链或其他结构解决冲突。负载因子是元素数除以桶数，超过阈值通常触发 rehash，重新分配桶并使迭代器失效。

### 60. 哈希冲突

主要方法是拉链法和开放寻址；开放寻址又有线性探测、二次探测和双重哈希。选择时考虑删除、负载因子、缓存局部性和最坏情况攻击。

### 61. unordered_map 退化

大量键落入同一桶时，查找会从均摊 O(1) 退化到 O(n)。原因可能是糟糕哈希、敌意输入、负载因子过高或键等价实现错误。

### 62. priority_queue

默认用 vector 保存完全二叉堆，父子关系可由索引计算。push 在尾部插入后上浮，pop 把末元素换到根后下沉，均为 O(log n)，top 为 O(1)。

### 63. 游戏中的栈和队列

栈用于状态返回、解析、撤销和 DFS；队列用于消息、事件、对象复用、任务调度和 BFS。答案应把数据结构的 LIFO/FIFO 性质与具体顺序需求对应起来。

### 64. 简化 vector

维护 `data/size/capacity`，用 allocator 申请未构造存储，placement construct 元素，析构有效区间。扩容时申请新块并 `move_if_noexcept`，成功后再销毁旧块以提供异常安全；实现拷贝/移动、reserve、push 和下标。

### 65. LRU Cache

哈希表把 key 映射到双向链表节点，链表头表示最近使用、尾表示最久未用。get/put 时把节点移到头，超容量删除尾并同步哈希表，所有操作均摊 O(1)。

## 算法解法骨架

### 66. 二分查找

使用半开区间 `[l,r)` 时循环条件为 `l<r`。找第一个 `>= target`：若 `a[mid] < target` 则 `l=mid+1`，否则 `r=mid`；精确查找在 lower_bound 结果处再判断相等。时间 O(log n)，空间 O(1)。

### 67. 反转链表

迭代维护 `prev、cur、next`，逐个把 `cur->next` 指向 prev；递归让后继反转后执行 `head->next->next=head` 并断开原 next。时间 O(n)，迭代空间 O(1)，递归栈 O(n)。

### 68. 链表环入口

快慢指针相遇后，将一指针放回头部，两者每次一步，再次相遇即入口。设头到入口 a、入口到首次相遇 b、环长 c，可由 `2(a+b)=a+b+kc` 推得 `a=(k-1)c+(c-b)`。

### 69. 合并有序链表

两个链表用哑节点和双指针每次接较小节点，O(m+n)。K 个链表可用大小 K 的最小堆，每取一个节点再压入其 next，时间 O(N log K)，空间 O(K)。

### 70. 第 K 大

维护大小 K 的最小堆，时间 O(n log K)、空间 O(K)；Quickselect 按枢轴分区，平均 O(n)、最坏 O(n²)，随机枢轴可降低退化概率。

### 71. Top-K 高频元素

先用哈希表计数，再用大小 K 的最小堆保存频率最高项，O(n log K)；频率最大不超过 n 时也可桶排序到 O(n) 时间和 O(n) 空间。

### 72. 两数与三数之和

两数之和用哈希表记录已见值，O(n)。三数之和先排序，固定一个数后双指针收缩，并跳过重复值，O(n²)。注意整数溢出和去重位置。

### 73. 最长无重复子串

滑动窗口维护字符最后位置或计数；右指针加入字符，出现重复时移动左边界到合法位置。每个字符至多进出一次，时间 O(n)，空间 O(字符集)。

### 74. 最小覆盖子串

用 need/window 计数和满足种类数 valid。右扩直到覆盖全部需求，再左缩寻找最短并更新答案；时间 O(n)，空间 O(字符集)。

### 75. 合并区间

按左端点排序，顺序扫描；当前区间与结果尾相交则扩展右端，否则追加新区间。时间 O(n log n)，额外空间取决于排序和结果。

### 76. 旋转数组查找

二分中至少一侧有序，判断 target 是否落在有序半区以舍弃另一半。无重复时 O(log n)；有大量重复时无法判断有序侧，最坏退化 O(n)。

### 77. 栈队列互相实现

两栈实现队列：输入栈接收，输出栈为空时整体倒入，均摊 O(1)。队列实现栈：push 后轮转旧元素，或 pop 时搬运到另一个队列，至少一种操作为 O(n)。

### 78. 有效括号与最小栈

括号题遇左括号入栈，右括号检查栈顶对应关系，最终栈空才有效。最小栈用辅助栈同步保存到当前位置的最小值，push/pop/top/getMin 均 O(1)。

### 79. 单调栈

维护尚未找到答案的索引，并保持值单调；新元素破坏单调性时弹栈并结算距离或面积。每个元素至多进出一次，时间 O(n)，空间 O(n)。

### 80. 二叉树遍历

递归按访问根的位置区分前中后序。迭代前序用栈先压右再压左；中序沿左链入栈；后序可用根右左后反转，或记录上次访问节点。均为 O(n)。

### 81. 层序、深度与直径

层序用队列按当前层大小处理。最大深度为 `1+max(left,right)`；直径在后序计算左右深度之和并更新全局最大，时间 O(n)。

### 82. 最近公共祖先

普通二叉树递归：当前为 null 或目标就返回；左右都非空则当前为 LCA，否则返回非空侧。BST 可利用值域从根向同一侧移动，分叉点即 LCA。

### 83. 验证 BST

中序遍历必须严格递增，或递归为节点传递允许的开区间上下界。不要只比较节点与直接子节点；时间 O(n)，递归空间 O(h)。

### 84. 岛屿与省份

对每个未访问陆地/节点启动 DFS 或 BFS，启动次数就是连通分量数。邻接矩阵省份题 O(n²)，网格岛屿题 O(mn)；也可使用并查集。

### 85. 拓扑排序

统计入度，把所有零入度节点入队；每取一节点就删除其出边并把新零入度节点入队。处理数等于节点数则无环，否则存在环，复杂度 O(V+E)。

### 86. 并查集

parent 表示集合森林，find 使用路径压缩，union 按秩或大小合并。m 次操作均摊接近 O(1)，严格为反 Ackermann 复杂度。

### 87. Dijkstra

对非负边维护 dist 和最小堆；弹出当前最小距离节点，若条目过期则跳过，再松弛所有出边。邻接表加堆为 O((V+E)log V)，存在负边时不能使用。

### 88. A*

优先扩展 `f(n)=g(n)+h(n)` 最小节点，g 是已知路径成本，h 是到目标估计。h 不高估真实代价时可保证最优；一致启发还能避免重复打开节点。h=0 时退化为 Dijkstra。

### 89. 0/1 与完全背包

0/1 背包每件只取一次，一维 DP 容量必须从大到小遍历；完全背包可重复取，容量从小到大遍历。时间 O(nC)，空间 O(C)。

### 90. 最长递增子序列

O(n²) DP 令 `dp[i]` 为以 i 结尾长度；优化法维护每个长度的最小结尾 tails，用 lower_bound 替换，时间 O(n log n)，tails 长度即答案。

### 91. 最长公共子序列

`dp[i][j]` 表示两个前缀的 LCS。字符相等取 `dp[i-1][j-1]+1`，否则取上或左最大，时间 O(mn)，空间可滚动到 O(min(m,n))。

### 92. 网格最短路与拆障碍

等权移动用 BFS。若进入空格代价 0、拆障碍代价 1，使用 0-1 BFS：0 权边压队首，1 权边压队尾，时间 O(V+E)；一般非负代价用 Dijkstra。

### 93. 最大人工岛

先 DFS/BFS 给每座岛编号并记录面积；再枚举每个 0，收集四邻不同岛编号，候选面积为 1 加这些面积之和。全为 1 时答案是总格数，复杂度 O(mn)。

### 94. 地下城游戏

从终点反向 DP，`dp[i][j]` 表示进入该格至少需要的生命，转移为 `max(1, min(下,右)-grid[i][j])`。边界设无穷哨兵并给终点外侧两个位置设 1，时间 O(mn)。

### 95. Trie 敏感词过滤

每个节点保存“字符到子节点”的映射和终止标记，插入与查询复杂度为 O(词长)。字符集不固定时使用 unordered_map 或压缩映射而不是固定 26 数组；扫描文本可从每个起点沿 Trie 匹配，规模更大时考虑 Aho-Corasick。
